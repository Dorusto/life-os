# Task: Migrate LLM backend to OpenAI-compatible API

## Context

Majordom currently calls Ollama via its native `/api/chat` endpoint.
We want to switch to the OpenAI-compatible `/v1/chat/completions` endpoint,
which is supported by Ollama, OpenRouter, DeepSeek API, Claude API, and others.

This unlocks cloud LLM providers (OpenRouter etc.) with zero code changes —
the user just sets env vars.

## Files to change

### 1. `backend/core/config/settings.py`

Rename `OllamaConfig` → `LLMConfig`. Change all env var names from `OLLAMA_*` to `LLM_*`.
Add `api_key` field. Keep the `settings.ollama` attribute name to avoid touching every call site — just rename the class and env vars.

```python
@dataclass
class LLMConfig:
    base_url: str = ""
    api_key: str = ""
    vision_model: str = ""
    chat_model: str = ""
    categorize_model: str = ""

    def __post_init__(self):
        self.base_url = os.getenv("LLM_BASE_URL", "http://ollama:11434")
        self.api_key = os.getenv("LLM_API_KEY", "")
        self.vision_model = os.getenv("LLM_VISION_MODEL", "qwen2.5vl:7b")
        self.chat_model = os.getenv("LLM_CHAT_MODEL", "qwen2.5:7b")
        self.categorize_model = os.getenv("LLM_CATEGORIZE_MODEL", self.chat_model)

    @property
    def model(self) -> str:
        return self.vision_model
```

In `Settings`, rename field: `ollama: LLMConfig = field(default_factory=LLMConfig)`.
Keep it as `settings.ollama` so call sites don't break.

Remove `TelegramConfig` and the Telegram validation from `Settings.validate()` — Telegram bot is removed from this project.

### 2. `backend/api/chat.py`

Switch from Ollama format to OpenAI-compatible format.

**Endpoint:** `{base_url}/v1/chat/completions`

**Auth header** (add only if `api_key` is set):
```python
headers = {"Content-Type": "application/json"}
if settings.ollama.api_key:
    headers["Authorization"] = f"Bearer {settings.ollama.api_key}"
```

**Request payload** — non-streaming (tool calls):
```python
payload = {
    "model": model,
    "messages": messages,
    "stream": False,
    "tools": TOOLS,
    "tool_choice": "auto",
    "options": {"temperature": 0.3, "num_predict": 512, "num_ctx": 8192},
}
```
Note: `"think": False` must be removed — it's Ollama-specific and causes errors on cloud APIs. `"options"` is also Ollama-specific; for cloud APIs it's silently ignored, so it's safe to keep.

**Response parsing** — OpenAI format:
```python
# was: response.get("message", {})
data = response.json()
assistant_message = data.get("choices", [{}])[0].get("message", {})
```

**Streaming** — OpenAI SSE format:
```python
# Each chunk: {"choices": [{"delta": {"content": "..."}}]}
# was: data["message"]["content"]
content = data.get("choices", [{}])[0].get("delta", {}).get("content", "")
```

Update `_stream_ollama_response` and `_call_ollama_non_streaming` accordingly.
Rename both functions to `_stream_llm_response` and `_call_llm`.
