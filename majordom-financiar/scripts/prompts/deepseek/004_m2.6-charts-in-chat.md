# M2.6 — Charts in Chat

## Context

Majordom is a personal finance assistant. The chat interface renders rich cards
(ProposalCard, ClarificationCard, etc.) when the agent returns structured JSON.
We want to extend this with inline charts — when the agent returns
`{"type": "chart", ...}`, the frontend renders a Recharts graph instead of text.

**Data sources (important):**
- Financial data (spending, budgets, income) → fetched from Actual Budget via
  existing backend tools: `get_spending_history`, `get_monthly_stats`,
  `get_budget_status`, `get_transactions`. Never query SQLite for financial data.
- Vehicle data (L/100km, odometer, per-vehicle breakdown) → SQLite `vehicle_log`
  table. This table is implemented in M3. For M2.6, only implement the frontend
  renderer and the financial chart tools — vehicle charts come in M3.

## Files to read first

- `frontend/src/pages/Chat.tsx` — how chunks are parsed and rendered
- `frontend/src/components/ProposalCard.tsx` — example rich card component
- `backend/api/chat.py` — system prompt and tool routing
- `backend/tools/registry.py` — existing tools list
- `backend/tools/` — existing tool implementations

## Tasks

### 1. Backend — new `get_chart_data` tool

Add a tool `get_chart_data` to the registry:

```json
{
  "name": "get_chart_data",
  "description": "Fetch time-series financial data for chart rendering. Use when the user asks for trends, charts, graphs, or visualizations of spending/budget over time.",
  "parameters": {
    "type": "object",
    "properties": {
      "metric": {
        "type": "string",
        "enum": ["spending_by_category", "spending_over_time", "budget_vs_actual"],
        "description": "What to chart"
      },
      "months": {
        "type": "integer",
        "description": "Number of past months to include (default: 6)"
      },
      "category": {
        "type": "string",
        "description": "Filter by category name (optional, for spending_by_category)"
      }
    },
    "required": ["metric"]
  }
}
```

The tool returns a JSON string with `{"type": "chart", "chart_type": "bar"|"line", "title": "...", "data": [...], "x_key": "...", "y_key": "..."}`.

Data is fetched from Actual Budget using the same pattern as `get_monthly_stats`
and `get_spending_history` — never from SQLite.

### 2. Backend — system prompt update

Add to `_build_system_prompt()` in `backend/api/chat.py`:

```
- To show spending trends, charts, or graphs: call get_chart_data, then return
  the result JSON directly without wrapping it in text.
```

### 3. Frontend — ChartCard component

Create `frontend/src/components/ChartCard.tsx`:
- Accepts `data`, `chart_type` ("bar" | "line"), `title`, `x_key`, `y_key`
- Renders `<BarChart>` or `<LineChart>` from Recharts (already in package.json)
- Same visual style as other cards (bg-surface, border-border, rounded-2xl)
- Responsive width (use `<ResponsiveContainer width="100%" height={200}>`)

### 4. Frontend — wire into Chat.tsx

In `handleChatChunk`, add detection for `parsed.type === 'chart'`:

```typescript
if (parsed.type === 'chart') {
  setMessages(prev => [...prev, {
    role: 'chart' as const,
    content: '',
    chart: parsed,
  }])
  return
}
```

Add `chart` to the `Message` interface and render `<ChartCard>` in the message
list, same pattern as `proposal`, `clarification`, etc.

## What NOT to do

- Do not query SQLite for financial data — all financial data comes from AB tools.
- Do not add vehicle charts (L/100km, odometer) — those are M3, after
  `vehicle_log` table exists.
- Do not split streaming in `api.ts` — keep raw chunk delivery, handle JSON
  detection in `handleChatChunk` only.
- Do not use `localStorage.getItem('token')` — always use `getToken()` from
  `frontend/src/lib/auth.ts`.

## Example flow

```
User: "show me my transport spending over the last 6 months"
  → LLM calls get_chart_data(metric="spending_over_time", months=6, category="Transport")
  → tool fetches from AB, returns chart JSON
  → LLM returns the JSON as its response
  → frontend detects type=chart → renders BarChart inline in chat
```
