# Task: Notifications appear in chat conversation

## Context

**Depends on M4.10** (persistent chat history) — the `chat_history` table must exist before this task.

Currently, daily digest and vehicle reminders are sent only as Web Push browser notifications.
The user wants these to also appear as messages in the chat conversation, so when they open
the chat they see Majordom's notification as a regular assistant message.

## How it works

When `run_daily_digest()` sends a push notification, it also saves the message content
to `chat_history` for all active users. When the user opens chat, the notification
appears as a Majordom message in the conversation timeline.

The push notification still fires (for when the user is not in the app).

## Backend — `backend/services/notification_service.py`

Add a helper function to save the notification body to chat history for all users:

```python
async def _save_to_chat_history(body: str, db: MemoryDB) -> None:
    """Save notification text to chat history for all active users."""
    import sqlite3, time
    conn = sqlite3.connect(db.db_path)
    try:
        # Get all distinct user_ids that have push subscriptions
        rows = conn.execute(
            "SELECT DISTINCT user_id FROM push_subscriptions"
        ).fetchall()
        user_ids = [r[0] for r in rows if r[0]]
        if not user_ids:
            return
        ts = int(time.time() * 1000)
        conn.executemany(
            "INSERT INTO chat_history (user_id, role, content, ts) VALUES (?, ?, ?, ?)",
            [(uid, "assistant", body, ts) for uid in user_ids],
        )
        conn.commit()
    finally:
        conn.close()
```

In `run_daily_digest()`, call this immediately after `push_svc.send_to_all(...)`:

```python
await push_svc.send_to_all(
    user_id="default",
    title="Majordom",
    body=body,
    url="/chat",
)

# Also save to chat history so it appears in the conversation
await _save_to_chat_history(body, db)
```

## What NOT to change

- Push notification flow (still fires as before)
- Frontend chat rendering (notification appears as a regular assistant message)
- Anti-spam guard logic
- The `body` format (already human-readable)

## Gotcha

The `chat_history` table might not exist if M4.10 hasn't been deployed yet.
