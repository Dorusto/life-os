# DeepSeek Prompt — Fuel Receipt Flow (M3.3)

## Context

Majordom is a personal finance assistant. The user's primary vehicle workflow is: **take photo of gas station receipt → review card → confirm → done**. On confirm, two writes happen:
1. AB transaction (financial: amount → "Car Costs" or "Motorbike Costs" category)
2. `vehicle_log` INSERT (operational: liters, ODO, consumption, location)

The existing receipt flow (grocery receipts) lives in `receipts.py` + `receipt_service.py` + `vision_engine.py`. The fuel flow **extends** this infrastructure — same upload endpoint, extended response, extended confirm.

**Vehicles in the system (active):**
- Toyota Corolla "Cora" — car
- Honda NC750X DCT "Wabi Sabi" — motorcycle

Vehicle detection: look at each vehicle's last recorded ODO, pick the one closest to the user-entered ODO.

```python
class FuelConfirmRequest(BaseModel):
    receipt_id: str
    # AB transaction fields
    account_id: str
    category_name: str     # e.g. "Car Costs", "Motorbike Costs"
    date: str              # YYYY-MM-DD
    station: str           # payee in AB
    total_eur: float
    # vehicle_log fields
    vehicle_id: int
    liters: float
    price_per_liter: Optional[float]
    odo_km: Optional[float]
    full_tank: bool = True
    missed_fill: bool = False
    fuel_grade: Optional[str] = None
    notes: Optional[str] = None

class FuelConfirmResponse(BaseModel):
    success: bool
    duplicate: bool
    transaction_id: Optional[str]
    vehicle_log_id: Optional[int]
    # post-confirm stats (shown in grey text after card)
    km_since_last: Optional[float]
    consumption_l100km: Optional[float]
    cost_per_km: Optional[float]

@router.post("/receipts/{receipt_id}/confirm-fuel", response_model=FuelConfirmResponse)
async def confirm_fuel_receipt(
    receipt_id: str,
    request: FuelConfirmRequest,
    current_user: str = Depends(get_current_user),
):
```

**Confirm logic (in order):**

1. Validate `receipt_id` matches (same check as existing confirm)
2. **AB transaction** — call `service.confirm()` with `merchant=request.station`, `amount=request.total_eur`, `category_id=request.category_name`, `account_id=request.account_id`, notes=`"[fuel] {request.liters}L"`
3. **vehicle_log INSERT** — via `db.insert_vehicle_log_entries([{...}])` with `source="manual"`, `fuelio_unique_id=None`
4. **Post-confirm stats** — query last valid fuel entry for this vehicle to calculate:
   - `km_since_last = request.odo_km - last_entry.odo_km` (if both exist)
   - `consumption_l100km = request.liters / km_since_last * 100` (only if `full_tank=True` and `missed_fill=False` and last entry was also `full_tank`)
   - `cost_per_km = request.total_eur / km_since_last`
5. Return `FuelConfirmResponse` with stats

**ODO validation** (do server-side too): if `km_since_last > 1500` → include `"odo_warning": true` in response (frontend already showed warning, but log it).

## 3. `backend/core/memory/database.py` — add method

```python
def get_last_odo_per_vehicle(self) -> list[dict]:
    """Return last recorded ODO for each active vehicle. Used for vehicle detection."""
    conn = self._get_conn()
    try:
        rows = conn.execute("""
            SELECT v.id, v.name, v.active,
                   MAX(vl.odo_km) as last_odo
            FROM vehicles v
            LEFT JOIN vehicle_log vl ON vl.vehicle_id = v.id AND vl.odo_km IS NOT NULL
            GROUP BY v.id
            ORDER BY v.active DESC, v.name
        """).fetchall()
        return [dict(r) for r in rows]
    finally:
        conn.close()
```

Also add helper for post-confirm stats:

```python
def get_last_fuel_entry(self, vehicle_id: int) -> dict | None:
    """Return the most recent full-tank fuel entry for consumption calculation."""
    conn = self._get_conn()
    try:
        row = conn.execute("""
            SELECT * FROM vehicle_log
            WHERE vehicle_id = ? AND entry_type = 'fuel' AND fuel_full_tank = 1 AND fuel_missed = 0
            ORDER BY date DESC LIMIT 1
        """, (vehicle_id,)).fetchone()
        return dict(row) if row else None
    finally:
        conn.close()
```

## Gotchas

1. **`receipt_service.py`**: `process_image()` returns a dict — add `receipt_type`, `liters`, `price_per_liter`, `fuel_grade` to the returned dict. Also call `db.get_last_odo_per_vehicle()` and compute `suggested_vehicle_id` when `receipt_type == "fuel"`.
2. **`VisionEngine`** parses the Ollama JSON response — ensure the parser handles the new fields (null-safe).
3. **`ReceiptData`** dataclass in `parser.py`: add optional fields `receipt_type`, `liters`, `price_per_liter`, `fuel_grade` OR bypass it and work directly with dicts in `receipt_service.py`.
4. **AB category lookup**: `service.confirm()` currently uses `category_id` (internal slug). The fuel card sends `category_name` (AB name like "Car Costs"). In `confirm_fuel_receipt()` endpoint, look up the category by name from AB, not by internal slug.
5. **`full_tank + missed_fill`**: L/100km is only meaningful when `full_tank=True AND missed_fill=False AND last entry was also full_tank`. If not, store the entry but set `consumption_l100km=None` in the response.
6. Post-confirm stats: query `db.get_last_fuel_entry(vehicle_id)` BEFORE inserting the new entry to get the previous ODO for km calculation.
