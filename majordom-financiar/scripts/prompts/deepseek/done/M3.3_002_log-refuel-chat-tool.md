# DeepSeek Prompt — log_refuel Chat Tool (M3 text flow)

## Context

Majordom supports two ways to log a refuel:
1. **Photo** (prompt 020, M3.3) — photo of receipt → FuelReceiptCard
2. **Text** (this prompt, M3.1) — "am alimentat 40L cu €70 din Shell Leiden, odo 33000km" → same FuelReceiptCard

Both flows show the **same** `FuelReceiptCard` component. The difference is how the card is triggered and which confirm endpoint it calls.

**Prerequisite:** prompt 020 must be implemented first (`FuelReceiptCard` component + `MemoryDB.get_last_odo_per_vehicle()` must exist).

## 1. `backend/tools/vehicle_proposals.py`

In-memory store — same pattern as `backend/tools/proposals.py`:

```python
"""In-memory store for pending refuel proposals (from chat text input)."""
import uuid

_refuels: dict[str, dict] = {}

def create(**kwargs) -> str:
    pid = uuid.uuid4().hex[:8]
    _refuels[pid] = kwargs
    return pid

def get(pid: str) -> dict | None:
    return _refuels.get(pid)

def delete(pid: str) -> None:
    _refuels.pop(pid, None)
```

Fields stored: `vehicle_id`, `vehicle_name`, `liters`, `total_eur`, `odo_km`, `location`, `full_tank`, `missed_fill`, `date`, `account_id`, `account_name`, `category_name`.

## 3. `backend/tools/finance/vehicle.py` — add `log_refuel`

```python
async def log_refuel(
    liters: float,
    total_eur: float,
    vehicle_name: str = "",
    odo_km: float | None = None,
    location: str = "",
    full_tank: bool = True,
) -> str:
    """
    Create a pending refuel proposal. Returns JSON with type='fuel_log'.
    Does NOT write to DB — only stores a pending proposal for frontend confirmation.
    """
    import json
    from datetime import date as _date
    from backend.tools import vehicle_proposals
    from backend.core.memory.database import MemoryDB
    from backend.core.config import settings
    from backend.tools.finance.actual_budget import _get_client

    today = _date.today().isoformat()
    db = MemoryDB(db_path=settings.memory.db_path)

    # Resolve vehicle from name or ODO proximity
    vehicles = db.get_last_odo_per_vehicle()
    vehicle_id = None
    display_name = vehicle_name

    if vehicle_name:
        matched = next((v for v in vehicles if vehicle_name.lower() in v["name"].lower()), None)
    elif odo_km is not None:
        # Pick vehicle whose last_odo is closest to odo_km
        active = [v for v in vehicles if v["active"]]
        matched = min(active, key=lambda v: abs((v["last_odo"] or 0) - odo_km), default=None)
    else:
        active = [v for v in vehicles if v["active"]]
        matched = active[0] if len(active) == 1 else None

    if matched:
        vehicle_id = matched["id"]
        display_name = matched["name"]

    # Resolve account
    account_id, account_name = "", ""
    try:
        accounts = await _get_client().get_accounts()
        if accounts:
            account_id = accounts[0].id
            account_name = accounts[0].name
    except Exception:
        pass

    # Default category based on vehicle type (heuristic: motorcycle names)
    category_name = "Motorbike Costs" if matched and any(
        kw in matched["name"].lower() for kw in ["wabi", "honda", "suzuki", "yamaha", "moto"]
    ) else "Car Costs"

    proposal_id = vehicle_proposals.create(
        vehicle_id=vehicle_id,
        vehicle_name=display_name,
        liters=liters,
        total_eur=total_eur,
        odo_km=odo_km,
        location=location,
        full_tank=full_tank,
        missed_fill=False,
        date=today,
        account_id=account_id,
        account_name=account_name,
        category_name=category_name,
    )

    price_per_liter = round(total_eur / liters, 3) if liters else None

    return json.dumps({
        "type": "fuel_log",
        "id": proposal_id,
        "vehicle_id": vehicle_id,
        "vehicle_name": display_name,
        "vehicles": vehicles,  # full list for dropdown
        "liters": liters,
        "total_eur": total_eur,
        "price_per_liter": price_per_liter,
        "odo_km": odo_km,
        "location": location,
        "full_tank": full_tank,
        "missed_fill": False,
        "date": today,
        "account_id": account_id,
        "account_name": account_name,
        "category_name": category_name,
    })
```

### Render FuelReceiptCard for fuel_log messages

**Reuse `FuelReceiptCard`** from prompt 020. The card needs a `ReceiptDraft`-like object — create an adapter:

```typescript
) : msg.role === 'fuel_log' && msg.fuelLog ? (
  <FuelReceiptCard
    draft={fuelLogToDraft(msg.fuelLog)}
    confirmEndpoint={`/vehicle/proposals/${msg.fuelLog.id}/confirm`}
    onConfirmed={(stats) => { /* show stats text */ }}
    onCancelled={() => { /* remove or mark cancelled */ }}
    onSwitchToGrocery={undefined}  // no grocery tab for chat-triggered refuels
  />
```

**`fuelLogToDraft()`** converts `FuelLogData` to the `ReceiptDraft` shape:
- `receipt_id = fuelLog.id`
- `image_url = ""` (no photo)
- `merchant = fuelLog.location`
- `amount = fuelLog.total_eur`
- `liters = fuelLog.liters`
- etc.

**OR** (simpler approach): extend `FuelReceiptCard` to accept either a `ReceiptDraft` or a `FuelLogData` via a union prop and handle both internally.

### Add `confirmEndpoint` prop to FuelReceiptCard (from prompt 020)

When implementing `FuelReceiptCard` in prompt 020, make the confirm endpoint configurable via prop:

```typescript
interface FuelReceiptCardProps {
  draft: ReceiptDraft | FuelLogData
  confirmEndpoint?: string  // defaults to `/receipts/${receipt_id}/confirm-fuel`
  onConfirmed: (stats: FuelConfirmResponse) => void
  onCancelled: () => void
  onSwitchToGrocery?: () => void
}
```

The card calls `confirmEndpoint` (or the default) on confirm. This way one component handles both flows.
