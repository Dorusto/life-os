# DeepSeek Prompt — Unify FuelReceiptCard (M3.3 refactor)

## Problem

`log_refuel` returns `FuelLogData` (different structure from `ReceiptDraft`), causing
`FuelReceiptCard` to have dual-mode logic (`isTextMode`, conditional rendering, missing fields).
Every new field requires patching both paths. This is tech debt — fix it now.

## Goal

`FuelReceiptCard` always receives `ReceiptDraft`. No dual-mode. No `FuelLogData`. No `isTextMode`.

---

## 1. `backend/tools/finance/vehicle.py` — log_refuel returns ReceiptDraft-compatible JSON

Return exactly these fields (same names/types as ReceiptDraft Pydantic model):

```json
{
  "type": "fuel_log",
  "receipt_id": "<proposal_id>",
  "receipt_type": "fuel",
  "merchant": "<location>",
  "amount": 73.69,
  "date": "2026-05-31",
  "suggested_category_id": "<category_name>",
  "category_source": "keywords",
  "categories": [{"id": "Transport", "name": "Transport", "emoji": "📦"}],
  "accounts": [{"id": "uuid...", "name": "ING current"}],
  "liters": 31.37,
  "price_per_liter": 2.349,
  "fuel_grade": null,
  "vehicles": [{"id": 3, "name": "Cora", "last_odo": 49800}],
  "suggested_vehicle_id": 3
}
```

Note: `receipt_id` contains the proposal UUID (used by frontend to call confirm endpoint).

---

## 2. `frontend/src/components/FuelReceiptCard.tsx` — single mode only

Props:
```typescript
interface FuelReceiptCardProps {
  draft: ReceiptDraft
  imageUrl?: string          // undefined in text mode (no photo)
  confirmEndpoint?: string   // defaults to /receipts/{receipt_id}/confirm-fuel
  onConfirmed: (stats: FuelConfirmResponse) => void
  onCancelled: () => void
  onSwitchToGrocery?: () => void  // only in photo mode
}
```

- Remove `FuelLogData` import
- Remove `isTextMode`, remove all `'id' in draft` / `'receipt_id' in draft` checks
- All fields read directly from `ReceiptDraft` (same as photo mode already does)
- `confirmEndpoint` prop: if provided, POST to that URL; otherwise POST to `/receipts/${draft.receipt_id}/confirm-fuel`
- Tab header: show only if `onSwitchToGrocery` is provided (no tabs in text mode)
- Photo thumbnail: show only if `imageUrl` is provided

---

## 3. `frontend/src/lib/api.ts`

- Remove `FuelLogData` interface entirely
- No other changes

---

## 4. `frontend/src/pages/Chat.tsx`

- `fuel_log` message role: store `draft: ReceiptDraft` (not `FuelLogData`)
  ```typescript
  fuelLog?: { draft: ReceiptDraft; fuelStats?: FuelConfirmResponse }
  ```
- When rendering `fuel_log` message: use `FuelReceiptCard` with:
  - `draft={msg.fuelLog.draft}`
  - `confirmEndpoint={/vehicle/proposals/${msg.fuelLog.draft.receipt_id}/confirm}`
  - NO `imageUrl`
  - NO `onSwitchToGrocery`
- Post-confirm: same `PendingFuelStatsDisplay` as photo mode

**Do NOT change**: `ReceiptDraft` interface, `FuelioImportCard`, `ReceiptCard` (grocery),
backend confirm endpoints (`/receipts/{id}/confirm-fuel`, `/vehicle/proposals/{id}/confirm`).
