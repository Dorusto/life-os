# Task: Persistent chat history + clear button

## Goal

Chat messages survive page refresh. History is server-side (SQLite),
so it's shared across devices (phone + PC see the same conversation).
The user can clear the history via a trash icon next to the help button.

## Backend

### New SQLite table

Add to `backend/core/db.py` (or wherever migrations run):

```sql
CREATE TABLE IF NOT EXISTS chat_history (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id   TEXT NOT NULL,        -- username from JWT (e.g. "doru")
    role      TEXT NOT NULL,        -- 'user' | 'assistant'
    content   TEXT NOT NULL,
    ts        INTEGER NOT NULL      -- unix timestamp, ms
);
CREATE INDEX IF NOT EXISTS idx_chat_history_user ON chat_history(user_id, ts);
```

### New API endpoints — `backend/api/chat_history.py`

```
GET  /api/chat/history          → returns last 100 rows for current user, oldest first
POST /api/chat/history          → appends one or more messages for current user
DELETE /api/chat/history        → deletes all rows for current user
```

All endpoints require `get_current_user` (same auth as `/api/chat`).
Use `current_user` (the username string) as `user_id` in all queries.

**GET response:**
```json
[{"role": "user", "content": "...", "ts": 1234567890000}, ...]
```

**POST body:**
```json
[{"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]
```
Inserts each message with `user_id = current_user` and `ts = now()`.
After insert, delete rows beyond the last 500 for this user:
```sql
DELETE FROM chat_history
WHERE user_id = ? AND id NOT IN (
    SELECT id FROM chat_history WHERE user_id = ? ORDER BY ts DESC LIMIT 500
)
```
Returns `{"saved": N}`.

**DELETE:** deletes all rows WHERE user_id = current_user. Returns `{"cleared": true}`.

Register router in `backend/main.py` with prefix `/api`.

### Context window limit in `backend/api/chat.py`

Before building the messages list for Ollama, truncate to last 10:

```python
# Limit history sent to LLM to avoid context overflow
MAX_CONTEXT = 10
history = req.messages[-MAX_CONTEXT:] if len(req.messages) > MAX_CONTEXT else req.messages

messages = [{"role": "system", "content": _build_system_prompt()}]
messages.extend([{"role": m.role, "content": m.content} for m in history])
```

Do NOT change anything else in chat.py.

### API calls — `frontend/src/lib/api.ts`

```typescript
export async function getChatHistory(): Promise<Message[]> {
  const res = await apiFetch('/api/chat/history')
  return res.json()
}

export async function saveChatHistory(messages: { role: string; content: string }[]): Promise<void> {
  await apiFetch('/api/chat/history', {
    method: 'POST',
    body: JSON.stringify(messages),
  })
}

export async function clearChatHistory(): Promise<void> {
  await apiFetch('/api/chat/history', { method: 'DELETE' })
}
```
