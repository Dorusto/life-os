## 2026-05-22 — Tombstone categories + payee bug investigation

### Resolved
- `get_monthly_stats()` now remaps tombstoned categories (same algorithm as `get_budget_status()`)
- Payee from receipt: `create_transaction(session, payee="Name")` works end-to-end — actualpy handles everything

### Files modified
- `backend/core/actual_client/client.py` — tombstone remap in `get_monthly_stats()`
- `backend/api/receipts.py` — payee passed as string to `create_transaction()`

### Lessons
- `docker compose restart` ≠ `docker compose build` — code baked in image doesn't change on restart.
- `print(..., flush=True)` > `logger.info()` for quick debug in containers without logging configured.
- `actualpy.create_transaction(session, payee="PayeeName")` works end-to-end with sync — no need to manually create Payees objects.
- actualpy change tracking: `before_flush` event → `session.info["messages"]` → `actual.commit()` → `sync_sync()`. If `group_id` is None, sync is not called.
- `transactions/description` in AB CRDT protocol = `payee_id` in SQLite (not the `notes` field).

### Unresolved
- Some old transactions (pre-fix) may lack payee in AB. Investigation exhausted — cost exceeded practical value. New transactions work correctly.
