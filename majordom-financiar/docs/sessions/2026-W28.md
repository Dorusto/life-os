## 2026-07-06 — #69: transfer+close atomic follow-up (closes the issue)

Direction depends on balance sign: positive drains out of the closing account, negative (a debt) gets paid off from the destination. - Implemented directly (not via DeepSeek) — touches provider pass-through + card conventions that aren't obvious from a prose spec alone.

Test data restored to original balances afterward (reopened both accounts, tombstoned the test transfer transactions).

Fixed by adding the bullet and trimming the tool's own overly-long description (4 quoted examples crammed into one field). Added as a permanent gotcha in `CLAUDE.md` so it's not a third strike.
