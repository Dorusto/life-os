### Lessons
- **`close_account` doesn't move money — see architecture.md rule 24.** Setting `Accounts.closed = True` on an account with balance ≠ 0 hides the money from the budget (verified: €1175 vanished from visible totals, restored on reopen) instead of forcing a transfer the way AB's native UI does. Current implementation is only safe for empty accounts.
- The confirmation-card mirror pattern (balance_adjustments as reference) is mechanical and reliable across ~7 layers; the only non-obvious layer the DeepSeek prompt missed was the `ActualBudgetProvider`/`FinanceProvider` pass-through (#126) — caught via CLAUDE.md's known-gotchas.
