### Lessons
- **`close_account` doesn't move money — see architecture.md rule 24.** Setting `Accounts.closed = True` on an account with balance ≠ 0 hides the money from the budget (verified: €1175 vanished from visible totals, restored on reopen) instead of forcing a transfer the way AB's native UI does. Current implementation is only safe for empty accounts.
- The confirmation-card mirror pattern (balance_adjustments as reference) is mechanical and reliable across ~7 layers; the only non-obvious layer the DeepSeek prompt missed was the `ActualBudgetProvider`/`FinanceProvider` pass-through (#126) — caught via CLAUDE.md's known-gotchas.

## 2026-07-06 — #69: transfer+close atomic follow-up (closes the issue)

### Resolved
- Implemented the deferred AB-parity flow from the note above: `client.close_account_with_transfer(account_id, destination_account_id)` — single `_get_actual()`/`commit()` doing transfer + close atomically (no split-failure window). Direction depends on balance sign: positive drains out of the closing account, negative (a debt) gets paid off from the destination. Reuses `_get_or_create_transfer_payee` (shared with `create_transfer()`) — no new money-movement code.
- Passed through all 3 layers (`client.py` → `actual_budget_provider.py` → `provider.py` Protocol, per #126 gotcha).
- `propose_close_account` now returns the full accounts list (mirrors `propose_account_transfer`); confirm endpoint requires `destination_account_id` when balance ≠ 0 (400 + proposal kept alive if missing, so the user can retry instead of restarting the whole flow); `CloseAccountCard.tsx` grew a destination selector (pattern borrowed from `AccountTransferCard.tsx`), confirm disabled until picked.
- Implemented directly (not via DeepSeek) — touches provider pass-through + card conventions that aren't obvious from a prose spec alone.
- Tested live end-to-end via the real chat endpoint (LLM tool call, not bypassed) on the local test stack: positive-balance case (`test`, €1175.78 → `BUNQ Savings`) and negative-balance case (`Cash`, -€47.48 → `ING current`) both zeroed the closing account and moved the balance correctly; missing-destination confirm correctly 400s without discarding the proposal. Test data restored to original balances afterward (reopened both accounts, tombstoned the test transfer transactions).
