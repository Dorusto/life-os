### Lessons
- `create_transfer()` in actualpy does NOT accept `imported_id` or `cleared` ‚Äî unlike `create_transaction()`. Any transfer-creation path needing dedup or reconciliation must set these directly on the returned `Transactions` objects (both legs have real, settable `financial_id`/`cleared` columns).
- Existing seeded test data in the dev AB instance already showed the exact bug pattern (transfer pairs with `cleared=0`, `financial_id=None`) ‚Äî useful as an unplanned before/after reference.
