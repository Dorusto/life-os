## 2026-06-21 — M4.2 + M4.3 + M4.4 Smart Alerts

### Resolved
- M4.2: Budget alert — immediate push after transaction if category exceeds monthly budget; anti-spam per category per day; `asyncio.ensure_future` fire-and-forget from tool layer
- M4.3: Income variance alert — daily digest warns if current month income < 80% of 3-month average (configurable threshold); skips if day < 10 or fewer than 2 months with data
- M4.4: Goal risk alert — weekly digest warns if savings goal with deadline is behind schedule (≤3 months + <90% or ≤6 months + <60%); anti-spam 7 days global
- `get_goals()` added to FinanceProvider Protocol + ActualBudgetProvider
- 3 new notification rules seeded on startup: `budget_alert`, `income_variance`, `goal_risk`

### Files modified
- `backend/core/finance/provider.py` — added `get_goals()` to Protocol
- `backend/core/finance/actual_budget_provider.py` — added `get_goals()` delegation
- `backend/services/notification_service.py` — `check_budget_alert()`, `_check_income_variance()`, `_check_goal_risk()` + wired into `run_daily_digest()`
- `backend/tools/finance/actual_budget.py` — `asyncio.ensure_future(check_budget_alert(...))` after successful tx
- `backend/main.py` — 3 new notification rule seeds

### Lessons
- `get_spending_history()` există în FinanceProvider protocol și ActualBudgetProvider, dar nu în ActualBudgetClient — metodă broken; M4.3 folosește `get_monthly_stats()` în loop ca workaround
- Anti-spam per-goal (`f"goal_risk_{id}"`) nu funcționează dacă digest-ul loghează global (`"goal_risk"`) — keys trebuie să se potrivească; fix: anti-spam global săptămânal

### Unresolved
- `get_spending_history()` în `ActualBudgetClient` lipsește — de implementat când e necesar
