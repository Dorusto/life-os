## 2026-06-01 — Notifications in chat + bug user_id="default"

### Resolved
- Daily digest now appears in chat history when user opens the app after receiving push notification
- `push_subscriptions` stores real `user_id` (from `current_user` auth), not hardcoded `"default"`
- `PushService.broadcast()` added: sends to ALL subscriptions without user_id filter (for system-wide digests)
- `window.addEventListener('focus', loadChatHistory)` in `App.tsx`: reloads chat when user returns to tab

### Files modified
- `backend/api/push.py` — `user_id=current_user` instead of `user_id="default"`
- `backend/core/memory/push_service.py` — `broadcast()` method added
- `backend/services/notifications.py` — `run_daily_digest` uses `broadcast()` + saves to `chat_history`
- `frontend/src/App.tsx` — `window focus` event listener for chat reload

### Lessons
- **Never hardcode `user_id="default"`** — always use `current_user` from auth. "Default" was a placeholder that broke user isolation everywhere it appeared.
- **Broadcast vs per-user push**: system notifications (daily digest) → `broadcast()` (all subscriptions). Personal alerts → `send_to_all(user_id=X)`.
- `window focus` for real-time sync: simpler than polling or WebSocket for cases where data changes rarely (daily notifications).
- Migration of existing subscriptions: `UPDATE push_subscriptions SET user_id='username' WHERE user_id='default'` via Docker exec.

## 2026-06-01 — Migration to OpenAI-compatible API + OpenRouter

### Resolved
- Chat service migrated from Ollama native `/api/chat` to OpenAI-compatible `/v1/chat/completions`
- OpenRouter integration: `LLM_BASE_URL=https://openrouter.ai/api` (no trailing `/v1`)
- `tool_calls[].function.arguments` parsed from string with `json.loads()` (OpenAI format)
- Current model config: `deepseek/deepseek-chat` (chat), `google/gemini-2.5-flash-lite` (vision)

### Files modified
- `backend/services/chat_service.py` — OpenAI-compatible streaming, `json.loads(args)` for tool arguments
- `backend/api/chat.py` — removed `num_ctx` from `options` (ignored by OpenAI-compatible endpoint)
- `.env.example` — updated model IDs and LLM_BASE_URL format

### Lessons
- **OpenAI format vs Ollama native**: `arguments` is a JSON **string** in OpenAI format, a **dict** in Ollama native. Always: `if isinstance(args, str): args = json.loads(args)`.
- **`LLM_BASE_URL` must NOT end with `/v1`**: code appends `/v1/chat/completions` automatically. `https://openrouter.ai/api/v1` → double `/v1` → 404.
- **Ollama `/v1/chat/completions` ignores `options.num_ctx`**: context window reverts to default (2048 tokens) → overflow with 20+ tool schemas. Use a cloud provider or Ollama's native endpoint.
- Performance: OpenRouter chat ~12s vs Ollama CPU-only ~4 minutes. Vision: instant vs 60s+.
- Model IDs change: `google/gemini-2.0-flash-lite` was deprecated on 2026-06-01. Current: `google/gemini-2.5-flash-lite`.

### Unresolved
- None.
